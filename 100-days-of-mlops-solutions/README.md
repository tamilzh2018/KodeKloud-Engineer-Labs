# 100 Days of MLOps — Solutions

Reference solutions for **[100 Days of MLOps](https://kodekloud.com/100-days-of-mlops)** by KodeKloud — a 100-day program that builds MLOps skills one focused challenge per day, from Python engineering foundations all the way to full end-to-end ML systems on Kubernetes.

Each day lives in its own folder containing a `solution.md` with a complete walkthrough: the concepts involved, the exact steps and commands, how to verify the result, and reference links.

## How it's structured

The program moves through a series of topic sections. Within and across them, the challenges progress through four levels of depth:

| Level | Scope |
|-------|-------|
| **L1 — Foundations** | A single tool or concept in a focused, self-contained task |
| **L2 — Intermediate** | Combining a few pieces into one working capability |
| **L3 — Advanced** | Multi-step, production-shaped scenarios |
| **L4 — Capstone** | End-to-end systems that integrate several tools together |

## Sections

| Section | Focus |
|---------|-------|
| **Python** | Engineering foundations: virtual environments, packaging, linting, pre-commit, project structure |
| **DVC** | Data & model versioning, remotes, and reproducible data pipelines |
| **MLflow** | Experiment tracking, the model registry, and model serving |
| **Training** | Reproducible training, evaluation, hyperparameter tuning, and training pipelines |
| **Feast** | Feature store: defining, materializing, and serving features |
| **Vault** | Managing secrets for ML workloads |
| **Data Quality** | Data validation with Great Expectations and quality gates in CI |
| **Docker** | Containerizing ML training and serving environments |
| **Serving** | Serving models with Flask, FastAPI, and BentoML; batch, A/B, and canary patterns |
| **Monitoring** | Observability with Prometheus, Grafana, and Evidently |
| **CI/CD** | Continuous integration and delivery for ML with Gitea Actions |
| **Pipelines** | Workflow orchestration with Argo Workflows and Prefect |
| **Kubernetes** | Deploying and serving ML on Kubernetes |
| **Capstone** | An end-to-end fraud-detection platform tying the whole program together |
